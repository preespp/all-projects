Embedded Systems Project

Project Description

This project focused on building a custom embedded Linux environment for the BeagleBone Black while also implementing kernel-level functionality on real hardware. The work covered both system bring-up and low-level programming, from kernel compilation and BusyBox configuration to GPIO control and interrupt-driven behavior.

In parallel, I collaborated on a kernel-level traffic light application that ran on actual hardware. The project gave me a much deeper understanding of how operating systems concepts connect to real embedded devices and real-time hardware interaction.

My Responsibilities

  • Built a custom Linux distribution for the BeagleBone Black, including Linux kernel compilation, BusyBox configuration, and QEMU emulation.
  • Developed Linux kernel modules for timers, asynchronous notifications, and a /proc interface for userspace monitoring and signaling.
  • Programmed real-time GPIO interaction for LEDs and pushbuttons using interrupt-driven kernelspace design.
  • Collaborated in a two-person team to implement and test a kernel-level traffic light control system on real hardware.

Skills I Learned

  • Embedded Linux bring-up, cross-compilation, and kernel debugging.
  • Kernel module development and low-level device interaction.
  • Interrupt-driven programming and real-time hardware control.
  • Serial communication and embedded toolchain workflows.
