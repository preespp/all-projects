Extending Site Reliability Engineering

Fault Mechanisms within ITBench

Fall 2025

ITBench is an open-source framework developed by IBM Research and UIUC for benchmarking the effectiveness of AI agents on real-world IT automation tasks.

This project was completed under mentorship from IBM engineers and focused on extending ITBench with more realistic fault scenarios for site reliability engineering.

The work involved cloud-inspired fault injection, Kubernetes-deployed microservices on a remote AWS cluster, and hands-on use of observability and infrastructure tooling.

My responsibilities
  • Contributed to an open-source research project under mentorship from IBM engineers
  • Implemented fault injection mechanisms inspired by real-world cloud incidents across AWS, GCP, Azure, and IBM Cloud
  • Developed application-level, container-level, and multi-service coordination fault scenarios within Kubernetes-deployed microservices on a remote AWS cluster
  • Worked hands-on with Docker, Kubernetes, Prometheus, Jaeger, OpenTelemetry, and Chaos Mesh tools with YAML automated via Ansible playbooks
  • Participated in Agile development practices including sprint planning, Git workflows, and peer code reviews
