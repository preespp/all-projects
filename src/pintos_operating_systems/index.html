Pintos Operating Systems Labs

Operating Systems, Fall 2025

I would present the Pintos lab sequence as one Operating Systems project rather than several separate entries, because the labs build on the same kernel codebase and progressively deepen core systems concepts.

The project sequence started with Lab 0 bootstrap work and then focused on Lab 1 Threads, Lab 2 User Programs, Lab 3a Virtual Memory, and Lab 3b Memory-Mapped Files. Lab 4 File Systems can be described as an optional extension if you want to include it.

The timeline fits well as a single senior-year project for Fall 2025, spanning September 3, 2025 through December 10, 2025, or through December 17, 2025 if you include the optional file system lab.

Recommended framing
  • Lab 0: Getting Real, released September 3, 2025, provided the Pintos bootstrap and environment setup
  • Lab 1: Threads, released September 17, 2025, focused on kernel thread scheduling
  • Lab 2: User Programs, released October 13, 2025, covered user programs and system calls
  • Lab 3a: Virtual Memory, released October 29, 2025, added demand paging
  • Lab 3b: Mmap Files, released November 9, 2025, implemented memory-mapped files
  • Optional Lab 4: File Systems, released November 16, 2025, extended the project into file-system work

    • Suggested portfolio bullets
  • Implemented core operating systems features in Pintos across kernel threads scheduling, user programs, system calls, virtual memory, and memory-mapped files
  • Worked in C on a teaching operating system codebase, strengthening understanding of kernel behavior, process execution, synchronization, and memory management
  • Built the project incrementally through a multi-lab sequence, using each lab to extend the same operating system with more advanced functionality
  • Optional: extended the system further with file-system functionality in Lab 4
